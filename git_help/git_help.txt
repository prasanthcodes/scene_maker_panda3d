full help:
-------------------------------------------

If the directory isn’t a Git repository and you want to make it one, run:
git init
This creates a .git folder, turning the directory into a Git repository.

Clone the repository (if you haven’t yet):
git clone https://github.com/prasanthcodes/scene_maker_panda3d
cd repository

Verify the repository:
git status
If it works without errors, you’re in a valid Git repository.

Once you’re in a valid Git repository, fetch and check out the remote branch as described earlier:
git fetch origin


List all remote branches to find the branch you want to check out:
git branch -r

Check out the remote branch and create a local tracking branch:
git checkout --track origin/<branch-name>

Alternatively, you can combine fetching and checking out in one step if you know the branch name:
git checkout <branch-name>
Git will automatically set up tracking if the branch exists on the remote.

Verify you're on the branch:
git branch

Add the files you want to commit:
git add .
This stages all modified files. To stage specific files, use:
git add <file-name>

Create a commit with a descriptive message:
git commit -m "Your commit message"

Push your changes to the remote branch on GitHub:
git push origin <remote-branch>

If the branch is new and doesn't exist on the remote, set the upstream branch:
git push --set-upstream origin <remote-branch>

to create new branch
git branch --set-upstream-to <remote-branch>

sets the default remote branch for the current local branch.

Any future git pull command (with the current local branch checked-out),
will attempt to bring in commits from the <remote-branch> into the current local branch.

git push -u origin <local-branch>

This automatically creates an upstream counterpart branch for any future push/pull attempts from the current <local-branch>. The upstream remote branch derived from the branch name - <local-branch> is also configured as the default for all subsequent actions like push/pull etc.




Run the following command to view the username and email set for your Git commits:
git config --global --list

Check the remote URL to see which GitHub account/repository is linked:
git remote -v

replace a local branch with a remote branch entirely in Git

    Make sure you've checked out the branch you're replacing 

    Assuming that master is the local branch you're replacing, and that "origin/master" is the remote branch you want to reset to:

git reset --hard main
git pull

fetch the branch locally:

git fetch origin <branchName>

move to that branch:

git checkout <branchName>

git pull is the combination of git fetch and git merge



to pull remote to local

git pull origin <branch-name>

to keep all files from remote

git checkout --theirs .
git add .
git commit

Note: Be cautious with this approach, as it discards your local changes in favor of the remote ones.


Verify and continue: If the pull was interrupted, you can retry the pull to ensure everything is up-to-date:
git pull origin <branch-name>


If you decide not to resolve conflicts and want to cancel the merge entirely:
git merge --abort
This will reset your working directory to the state before the pull attempt, discarding any merge attempts.
